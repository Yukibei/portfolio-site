## 1. 定位

**一句话：能把 AI 能力做成可上线产品的全栈工程师。**

两类读者：
- HR（30 秒）：Hero 区看完定位 + 硬数字 + 上线链接 → 转发技术面
- 技术面试官（3-5 分钟）：项目详情页看架构、演示、技术决策 → 形成"能干活"判断

原则：**首屏 30 秒讲清你是谁，详情页经得起 5 分钟拷打。**

### 内容要点

**Hero 硬数字候选**：mAP 91.61% / Rank-1 94.40%、2 个上线项目、12 个业务模块、Web+App 双端 Agent。

**项目详情页统一结构**（每页都按此模板，方便面试官扫读）：
1. 一句话定位 + 我的角色 + 时间
2. 演示（录屏视频/GIF 置顶）
3. 架构图
4. 3-4 个关键技术决策（"为什么这么做"——面试拷打点前置自答）
5. 量化结果
6. 链接（线上地址 / GitHub）

**AI 实践区**（王牌差异化，普通应届生作品集没有这块）：
- Pipeline Chain：一句话触发跨页面流水线（录屏）
- 小龙虾悬浮 Agent：一句话操作 App（录屏）
- 我的 AI 工作流：Claude Code + Agent 团队协作开发方法论（配工作截图）

**交互优化案例**（2-3 个，候选）：
- AI 操作卡片的状态反馈设计（执行中/确认/降级）
- 长任务异步轮询的用户感知优化（进度、骨架屏）
- 图搜话术优化（"未知球员"→"编号 XXX"+低置信度提示）

## 3. 视觉方向：篮球 × AI

- **配色**：深色底（近黑）+ 篮球橙 (#F97316 系) 主强调 + 热图渐变（蓝→青→黄→红）做数据可视化点缀
- **图形语言**：球场地板线框做分区装饰；投篮抛物线作为滚动引导线/章节过渡动效；ReID 检测框（角标框）做图片 hover 效果；等宽字体点缀数据
- **动效**（Framer Motion）：入场视差、数字滚动计数、抛物线路径动画；克制原则——每屏最多一个主动效
- **底线**：创意服务于叙事，不堆砌；移动端体验与桌面同权重（HR 大概率手机打开）

## 4. 技术架构

```
P1/P2：纯静态
Next.js (output: 'export') → out/ → ECS Nginx 静态托管
                                     server_name me.hooppupil.me

P3：+ 轻后端
FastAPI 小服务（复用 ReflexLearn：LiteLLM + RAG + SSE）
Nginx 反代 /api/* → FastAPI (新端口，如 8010)
知识库：简历 + 三个项目文档 + AI 实践文章 → 向量化
```

技术细节约定：
- 录屏一律转 **WebM/MP4 + `<video autoplay muted loop>`**，禁用大体积 GIF；懒加载
- 图片 WebP + next/image（静态导出用 unoptimized 模式 + 预压缩）
- 移动端断点优先设计；Lighthouse 移动端 ≥ 90 分作为 P2 验收线
- SEO：每页 metadata + OG 卡片（微信/钉钉转发简历链接时显示卡片，重要）
